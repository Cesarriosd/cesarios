Articles

Graphs

Graph Neural Networks for Financial Fraud Detection: A Review

Abstract: This review explores the role of Graph Neural Networks (GNNs) in addressing fraud detection and proposes a unified framework that categorises existing GNN methodologies applied to this field. Specifically, examining 4 main architectures: Graph Convolutional Networks (GCNs), Graph Attention Networks (GANs) ,Graph Temporal Networks (GTNs) and Heterogenous Graph Neural Networks.

Retrieval-Augmented Generation with Knowledge Graphs for Customer Service Question Answering

Abstract: In this article the authors propose the usage of Large Language Models and Knowledge graphs to improve RAG. The main problems that the authors (LinkedIn) have to tackle are: Compromised Retrieval Accuracy from Ignoring Structures and Reduced Answer Quality from Segmentation. They leverage knowledge graph capabilities using two types of information: Intra-ticket tree parsing (structure of the ticket itself) and inter-ticket connection (relation between different tickets), with this information the LLM is fed to create a better final answer. With this approach they reduce time of response from 40 hrs to 15 hrs (mean).

Computer Vision

SAMURAI: Adapting Segment Anything Model for Zero-Shot Visual Tracking with Motion-Aware Memory

Abstract: This paper proposes an adaptation of SAM 2, specifically designed for visual object tracking. SAMURAI operates in real time and incorporates a streaming memory architecture which enables it to process video frames sequentially while maintaining context over long sequences. The method proposed incorporates two key advancements: 1) a motion modeling system that refines the mask selection, enabling more accurate object position prediction in complex scenarios, and 2) an optimized memory selection mechanism that leverage a hybrid scoring system, combining the original mask affinity object and motion scores to retain more relevant historical information.

Time Series

Forecasting at Scale

Abstract: This article presents “Prophet”: an additive regression model with four main components: 1) a piecewise linear or logistic growth curve trend; 2) a yearly seasonal component modelled using Fourier series; 3) a weekly seasonal component using dummy variables; and 4) a user-provided list of important holidays.

Some remarkable points are that Prophet automatically detects changes in trends by selecting changepoints from the data and the main objective of this procedure is to face the problem of generating reliable time series forecasts at scale, this means “forecasting a large number and a variety of time series”.
